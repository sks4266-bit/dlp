# 맥체인 캘린더 UX 개선 정리

이 문서는 **맥체인 캘린더(Mcheyne Calendar)** 화면의 UX 개선 사항을 “사용자 관점(무엇이 좋아졌는가)”과 “개발자 관점(어디가 바뀌었는가 / 어떻게 동작하는가)”로 정리합니다.

- 주요 화면: `apps/web/src/pages/McheyneCalendarPage.tsx`
- 연관 화면: `apps/web/src/pages/McheyneTodayPage.tsx`, `apps/web/src/pages/HomePage.tsx`
- 연관 API(백엔드): `apps/api/src/routes/mcheyne.ts`, `apps/api/src/routes/mcheyne_text.ts`, `apps/api/src/routes/mcheyne_progress.ts`

---

## 1) 핵심 UX 목표

1. **오늘/해당 날짜에 빠르게 접근**: “오늘”을 눈에 띄게 강조하고, 1~2번 탭으로 본문을 열 수 있게
2. **본문 접근 동선 단축**: 바텀시트에서 각 reading을 즉시 열기(‘바로 열기’) 제공
3. **진행률 입력 마찰 최소화**: 체크/일괄 완료를 빠르고 확실하게(즉시 반응 + 실패 처리)
4. **월 이동의 가독성/조작성 강화**: 이전/다음 버튼으로 월 이동을 직관적으로

---

## 2) 캘린더(그리드) 영역 개선

### 2.1 ‘오늘’ 셀 자동 강조
- 오늘 날짜 셀에 **강조 테두리/그림자 + ‘오늘’ 배지(pill)**가 표시됩니다.
- 목표: 사용자가 캘린더에서 현재 위치를 즉시 인지.

### 2.2 오늘 셀 ‘바로가기’ 버튼
- 오늘 셀 내부에 **‘바로가기’** 버튼을 추가했습니다.
- 동작:
  - 로그인 상태: 해당 날짜 읽기 페이지로 이동 (`/mcheyne-today?month=..&day=..`)
  - 비로그인 상태: 로그인으로 이동(복귀 경로 포함)

> 구현 메모: 버튼 중첩 문제를 피하기 위해, 날짜 셀을 `<button>`에서 `role="button"`을 가진 `<div>`로 변경하고 키보드(Enter/Space)도 처리합니다.

---

## 3) 월 이동 UX 개선

### 3.1 이전/다음 월 버튼
- 월 선택 드롭다운 옆에 **이전(←) / 다음(→)** 버튼을 추가했습니다.
- 월은 1~12를 순환하도록 처리합니다.

### 3.2 ‘오늘’ 이동 버튼
- 월이 달라도 **‘오늘’ 버튼**으로 즉시 오늘 월로 이동하며,
- 이동 후 바텀시트를 자동으로 열어(옵션) 오늘 상세를 바로 확인할 수 있습니다.

---

## 4) 바텀시트(날짜 상세) UX 개선

### 4.1 날짜 상세 정보 표시
- 선택한 날짜의:
  - `월/일`
  - 완료 수 `완료: N/4`
  - reading1~4 목록
을 한 화면에 보여줍니다.

### 4.2 4개 본문 각각 ‘바로 열기’ 제공
- 바텀시트에 **“본문 바로 열기”** 섹션을 추가했습니다.
- reading1~4 각각 버튼 제공:
  - `1번 바로 열기 · <ref>`
  - `2번 바로 열기 · <ref>`
  - `3번 바로 열기 · <ref>`
  - `4번 바로 열기 · <ref>`
- 클릭 시 성경 본문 페이지로 이동:
  - `/bible?ref=<readingX>`

### 4.3 ‘이 날 본문 읽기’ 버튼
- 바텀시트 하단에 **‘이 날 본문 읽기’** 버튼을 제공:
  - 로그인: `/mcheyne-today?month=..&day=..`
  - 비로그인: 로그인으로 이동(복귀 경로 포함)

---

## 5) 진행률(체크) UX 개선

### 5.1 개별 체크(1~4)
- 로그인 사용자에 한해, 바텀시트에서 reading 1~4를 체크할 수 있습니다.
- 비로그인 사용자에게는 안내 문구를 표시합니다.

### 5.2 ‘일괄 완료’(4개 모두 읽음)
- 바텀시트의 reading1~4 영역에 **‘일괄 완료 (4개 모두 읽음)’** 버튼을 제공합니다.

#### UX 동작(추천 설계)
- 버튼 클릭 즉시:
  1) **체크박스 4개가 즉시 체크(Optimistic UI)**
  2) 체크 영역에 **짧은 팝/하이라이트 애니메이션**
  3) 하단에 **토스트**
- 저장 성공 시:
  - **바텀시트를 자동으로 닫음**
  - 캘린더 그리드의 해당 날짜 셀 색상이 **즉시 4/4 완료 색으로 반영**
- 저장 실패 시:
  - **Optimistic update를 롤백(원복)**
  - 바텀시트는 유지(재시도 가능)
  - 실패 토스트 표시

#### 기술 동작 요약
- `setDayDone(day, patch)`
  - optimistic update → API 저장 → server truth 재동기화
  - 실패 시 스냅샷으로 롤백
- `bulkCompleteDay(day)`
  - `setDayDone(day, {done1..4:1})` 호출
  - 성공 시 `setSheetOpen(false)`로 자동 닫기

## 6) 로그인/비로그인 정책

| 기능 | 로그인 필요 | 비로그인 시 동작 |
|---|---:|---|
| 월/일 계획(plan) 보기 | ❌ | 그대로 사용 가능 |
| 월/일 진행률(prog) 표시 | ✅ | 진행률 숨김(또는 안내 문구) |
| 체크(1~4) | ✅ | 안내 문구 표시 |
| 일괄 완료 | ✅ | ‘로그인 필요’ 버튼/유도 |
| 이 날 본문 읽기(텍스트) | ✅ | 로그인으로 유도(복귀 경로 포함) |

## 7) 테스트 체크리스트(수동 QA)

### 7.1 비로그인
- [ ] 캘린더 진입 시 plan이 정상 표시됨
- [ ] 오늘 셀이 강조되고 ‘바로가기’가 보임
- [ ] 오늘 ‘바로가기’ 클릭 시 로그인으로 이동(복귀 경로 포함)
- [ ] 바텀시트에서 ‘본문 바로 열기’는 작동(성경 페이지 이동)
- [ ] 체크/일괄 완료는 로그인 유도 문구/버튼이 보임

### 7.2 로그인
- [ ] 날짜별 완료 수(0~4)가 셀에 표시됨
- [ ] 체크 1~4 토글이 저장되고 즉시 UI 반영됨
- [ ] 일괄 완료 클릭 시: 체크 4개 즉시 체크 + 애니메이션 + 토스트
- [ ] 일괄 완료 성공 시 바텀시트가 자동으로 닫히고, 해당 날짜 셀 색상이 즉시 완료색으로 바뀜
- [ ] 실패(네트워크/서버 오류) 시 롤백되는지 확인

---

## 8) 추후 개선 아이디어(선택)

1. **완료된 셀을 탭했을 때 바텀시트에서 “완료 취소(전체 해제)”** 액션 제공
2. 바텀시트에서 ‘바로 열기’를 누른 뒤, 돌아왔을 때 **기존 바텀시트 유지/복원** 옵션
3. 캘린더에서 날짜 셀에 **작은 “열기” 아이콘(↗)**을 모든 날짜에 제공(현재는 오늘 셀만)
4. 완료/미완료 셀의 색 대비(접근성) 튜닝
